Подкласс может иметь и свои собственные свойства и методы. Давайте определим класс Scania и унаследуем его от того же класса Automobile:

class Automobile
  def initialize(model, year)
    @model = model
    @year = year
  end
  def info
    puts "Модель #{@model} #{@year} года выпуска"
  end
end

class Ferrari < Automobile
end

class Scania < Automobile
  attr_accessor :type
  def info
    puts "#{@type} автомобиль"
  end
end

r730 = Scania.new("R730", 2004)
r730.type = "Грузовой"
r730.info
# выведет: Грузовой автомобиль

                  
В вышеприведенном коде подкласс Scania наследует от суперкласса Automobile. Он имеет собственную переменную экземпляра type, а также определяет свой собственный метод info. Такая ситуация, когда в подклассе мы изменяем поведение метода из суперкласса, называется переопределением метода.

Когда мы ранее вызывали метод info для объекта класса Ferrari, вызывался метод суперкласса потому, что Ferrari не переопределял его.
А объект класса Scania вызвал свой собственный метод info, потому что он был переопределен.
