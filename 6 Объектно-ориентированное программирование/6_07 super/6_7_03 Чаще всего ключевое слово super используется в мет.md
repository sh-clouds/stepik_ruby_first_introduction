Чаще всего ключевое слово super используется в методе initialize.
Например, наш суперкласс имеет метод initialize, который принимает один аргумент и инициализирует переменную экземпляра:

class Automobile
  def initialize(name)
    @name = name
  end
end

                  
Теперь нам нужен подкласс Truck, который будет иметь дополнительную переменную экземпляра @trailer, и нам нужно определить свой собственный метод initialize. Вместо того, чтобы повторяться и задавать переменную экземпляра name в классе Truck, мы можем использовать его суперкласс с методом super следующим образом:

class Truck < Automobile
  def initialize(name, trailer)
    super(name)
    @trailer = trailer
  end
  def info
    if @trailer == true 
        puts "У автомобиля #{@name} есть полуприцеп"
    else
        puts "У автомобиля #{@name} нет полуприцепа"
    end
  end
end

                  
В примере выше мы передаем один из аргументов методу super, который в свою очередь вызывает метод initialize класса Automobile и устанавливает значение переменной экземпляра @name. А второй аргумент уходит в переменную @trailer.

Теперь мы можем создать новый объект и вывести информацию о нем:

daf = Truck.new("DAF", true)
daf.info

# выведет:
# У автомобиля DAF есть полуприцеп

                  
В примере мы использовали super для простого задания. Представьте себе наличие сложной программы, в которой выполняются сложные вычисления и операции. Вот тут-то и появляется реальная польза от принципа "не повторяйте код", и вызов super  является одним из способов его достижения.
