Можно, но не нужно

Как вы могли видеть во всех примерах этого урока, мы всегда передавали блок case переменную:

case x

                  
И затем в каждом операторе when сравнивали ее с конкретным значением (или значениями).

Однако в Ruby есть возможность использовать и другой вариант этого блока, когда переменная определяется заранее, не передается в case, и при этом в операторах when значение переменной сравнивается не с конкретным значением, а с результатом вычисления выражения:

temperatura = 3
case
when temperatura > 0 
  puts "Больше нуля"
when temperatura < 0 
  puts "Меньше нуля"
else
  puts "Равна нулю"
end

# выведет: Больше нуля

                  
Однако такой вариант не рекомендуется использовать. Более того, будьте внимательны и не путайте этот вариант с тем, который мы разбирали ранее. То есть, если вы используете переменную в case, то в операторах when используйте только точные значения, а не выражения. Иначе весь блок будет работать совсем не так, как вам может показаться.
