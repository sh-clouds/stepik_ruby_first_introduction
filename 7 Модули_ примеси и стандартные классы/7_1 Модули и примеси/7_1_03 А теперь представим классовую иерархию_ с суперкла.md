А теперь представим классовую иерархию, с суперклассом Animal и подклассами: Lion, Cow, Bird, Butterfly, Fish.

Все они имеют некоторый общий функционал, который они унаследовали от класса Animal, но есть и некоторые отличия, например, Bird и Butterfly могут летать, а Fish - плавать.

Вместо того, чтобы создавать отдельные методы для полетов и плавания для отдельных классов, можно создать отдельный модуль для полетов и отдельный для плавания и затем включить их (используя ключевое слово include) в нужные классы:

module Flying
  def fly
    puts "Я могу летать!"
  end
end

module Swimming
  def swim
    puts "Я могу плавать!"
  end
end

class Animal
end

class Lion < Animal
end

class Bird < Animal
  include Flying
end

class Butterfly < Animal
  include Flying
end

class Fish < Animal
  include Swimming
end

                  
Теперь объекты классов Bird и Butterfly могут летать, объекты класса Fish - плавать, а вот объекты других классов будут лишены таких возможностей:

eagle = Bird.new
eagle.fly
# выведет: Я могу летать!

carp = Fish.new
carp.swim
# выведет: Я могу плавать!

                  
Это называется подмешивание модулей. Ведь модули не позволяют создавать на их основе объекты, поэтому, чтобы использовать включенные в модули методы, можно подмешивать их в классы, что позволит использовать методы на объектах классов.
