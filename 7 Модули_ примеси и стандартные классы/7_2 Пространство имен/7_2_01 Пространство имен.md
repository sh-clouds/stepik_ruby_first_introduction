Пространство имен

Пространство имен — это именованный фрагмент программы, содержащий константы, методы, классы и модули.

Пространство имен организовывает код проекта в иерархию, напоминающую файловую систему. Как файлы с одинаковыми именами изолированы, если находятся в разных каталогах, так и классы, методы и константы Ruby изолированы в разных пространствах имен. Это позволяет избегать конфликтов между библиотеками и модулями, а также облегчает поиск и загрузку классов. В Ruby пространство имен организуется при помощи вложенных модулей и классов.

Например:

module Samsung
  class Laptop
    def name
      puts "Galaxy Book 2"
    end
  end

  class Phone
    def name
      puts "Galaxy S24"
    end
  end
end

module Apple
  class Laptop
    def name
      puts "MacBook Pro 14"
    end
  end
end

                  
Мы определили модуль Samsung, который группирует два класса, Laptop и Phone. Также создали модуль Apple, который тоже содержит класс Laptop. Но благодаря пространству имен, у нас могут существовать одновременно два класса с одинаковым названием, но в разных модулях.

Теперь мы можем вызывать классы в модуле используя оператор разрешения области видимости ::. Слева от оператора указывается имя модуля, справа — класс в модуле:

s = Samsung::Laptop.new
a = Apple::Laptop.new

s.name # выведет: Galaxy Book 2
a.name # выведет: MacBook Pro 14

                  
Преимущества пространства имен:
- нам становится легко распознавать родственные классы в нашем коде;
- это снижает вероятность конфликтов наших классов с другими одноименными классами. Мы можем иметь одинаковые имена классов в различных модулях.
