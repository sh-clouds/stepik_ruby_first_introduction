Proc очень похожи на методы. Они выполняют операции и могут принимать параметры.
Но по-настоящему мощным и уникальным Proc делает способность передавать их в методы, потому что Proc на самом деле являются объектами (как и всё в Ruby, как вы, надеюсь, помните).

Например:

hello = Proc.new do |x|
  puts "Привет, #{x}"
end

bye = Proc.new do |x|
  puts "До новых встреч, #{x}"
end

def say(arr, proc)
  arr.each { |x| proc.call x}
end

                  
Мы определили два Proc и метод, который принимает в качестве параметров массив arr и proc. Для каждого элемента массива он вызывает proc. Теперь hello и bye - это объекты, которые содержат соответствующие блоки кода.

Мы можем вызвать метод say и передать наши объекты Proc в качестве параметров:

humans = ["Михаил", "Дмитрий", "Иван"]
say(humans, hello)
say(humans, bye)

# выведет:
# Привет, Михаил
# Привет, Дмитрий
# Привет, Иван
# До новых встреч, Михаил
# До новых встреч, Дмитрий
# До новых встреч, Иван

                  
Мы можем передавать в наши методы столько Proc в виде аргументов, сколько захотим.

Использование Proc дает дополнительную гибкость, позволяя повторно использовать блоки кода более чем в одном месте без необходимости каждый раз печатать их. Proc, в основном, берут блоки кода и встраивают их в объект, что позволяет повторно использовать их и передавать по кругу.
